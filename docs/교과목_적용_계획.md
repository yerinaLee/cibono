# Cibono 교과목 적용 계획

## 목적

Cibono는 냉장고 재고, 유통기한, 영수증/주문내역 스캔, 레시피 추천, 할인 알림을 결합한 식재료 관리 앱이다.
한국방송통신대학교 컴퓨터과학과에서 배운 내용을 실제 서비스 기능, 데이터 구조, 보안, 운영 환경에 연결해 경진대회 출품 시 전공 지식 활용도를 명확히 보여주는 것을 목표로 한다.

## 현재 프로젝트 핵심 기능

- 냉장고 재고 등록, 수정, 삭제
- 식재료별 유통기한 관리
- 영수증/주문내역 이미지 기반 식재료 자동 추출
- 보유 재료와 유통기한을 고려한 레시피 추천
- 할인 정보와 사용자 가격 기준을 비교한 알림 생성
- Spring Boot API, PostgreSQL, Expo/React Native 앱 구조
- Docker Compose 기반 PostgreSQL 실행 환경

## 우선 적용 포인트

### 1. 교과목 적용 증빙 문서화

- `교과목 / 배운 개념 / 프로젝트 적용 기능 / 관련 코드 / 시연 방법` 형식으로 정리한다.
- 발표 자료와 README에 같은 표를 요약해 심사자가 전공 개념 적용 여부를 빠르게 확인할 수 있게 한다.
- 기능 구현 후 코드 위치와 화면 캡처를 계속 갱신한다.

### 2. 추천 알고리즘 고도화

- 유통기한 임박 재료를 우선 처리하는 우선순위 큐 개념을 적용한다.
- 재료와 레시피를 그래프 또는 집합 관계로 모델링한다.
- 추천 점수 공식을 명시한다.
  - 예: `보유 재료 점수 + 임박 재료 가중치 - 부족 재료 패널티 - 조리시간 패널티`
- 추천 결과에 "왜 추천됐는지"를 함께 표시한다.
  - 예: `계란 D-1, 우유 D-2 포함 / 부족 재료 1개`

### 3. 데이터베이스 정규화 및 인덱스

- `item` 마스터 테이블을 추가해 식재료 표준명을 관리한다.
- `item_alias` 테이블을 추가해 "대파", "쪽파", "파"처럼 같은 계열의 재료명을 매핑한다.
- `inventory`, `deal`, `price_alert`, `alert_event`에 조회 패턴 기반 인덱스를 추가한다.
- 알림 중복 생성을 막기 위해 `alert_event(user_id, deal_id)` unique 제약을 검토한다.
- ERD와 정규화 근거를 문서화한다.

### 4. 인증과 보안 강화

- 회원가입/로그인 API를 추가한다.
- 비밀번호는 BCrypt로 해시한다.
- JWT 기반 인증을 적용한다.
- 사용자별 데이터 격리 테스트를 추가한다.
- Gemini API key 등 민감 정보는 secret 파일 또는 환경 변수로 분리한다.
- 이미지 업로드 크기 제한, MIME 검증, 입력값 검증 DTO를 추가한다.

### 5. 영수증 이미지 처리 기능 강화

- 이미지 업로드 전 리사이즈/압축을 적용한다.
- 스캔 결과를 사용자가 수정 후 확정하는 흐름을 유지한다.
- 추출 결과에 신뢰도 또는 검토 필요 표시를 추가한다.
- 이미지 처리 전후 파일 크기와 처리 시간을 기록해 멀티미디어시스템 과목 적용 근거로 활용한다.

### 6. 백그라운드 작업과 알림 처리 개선

- 할인 알림 스캔을 스케줄러에서 주기적으로 실행한다.
- 알림 생성 로직을 트랜잭션으로 보호한다.
- 같은 할인 정보로 중복 알림이 생성되지 않도록 한다.
- 추후 작업 큐 구조를 도입할 수 있도록 스캔 작업 상태를 저장하는 테이블을 검토한다.

### 7. 로컬/NAS 운영 인프라 구성

- Docker Compose에 API, DB, reverse proxy를 포함한다.
- NAS 또는 로컬 디스크에 PostgreSQL 백업을 주기적으로 저장한다.
- Spring Boot Actuator, Prometheus, Grafana로 상태 모니터링을 구성한다.
- Caddy 또는 Nginx로 HTTPS reverse proxy를 구성한다.
- 외부 접속은 Tailscale VPN 또는 Cloudflare Tunnel을 검토한다.

담당자 B가 발표에서 강조할 메시지는 다음과 같다.

- 재고와 추천 기능은 식재료 데이터를 집합, 관계, 우선순위로 모델링해 구현했다.
- 영수증 이미지는 멀티미디어 데이터로 보고 압축, 검증, AI 추출, 사용자 확정 절차를 거친다.
- 추천 결과는 단순 AI 응답이 아니라 설명 가능한 점수 공식과 자료구조 기반 정렬로 제공된다.

### 담당자 간 연결 지점

Deals/Alerts와 Inventory/Recommend는 독립 기능처럼 보이지만, 출품 시에는 아래 연결 지점을 강조한다.

- `item` 표준명은 재고, 추천, 할인, 알림 규칙에서 함께 사용한다.
- Inventory의 보유 재료와 Deals의 할인 품목을 연결해 "부족 재료 할인 추천"으로 확장할 수 있다.
- Recommend의 부족 재료 목록을 Alerts Rules와 연결해 부족 재료가 할인되면 알림을 받게 할 수 있다.
- 공통 사용자 인증과 권한 검증을 통해 모든 데이터가 사용자별로 분리된다.
- 공통 인프라에서 API, DB, 스케줄러, 백업, 모니터링이 함께 운영된다.

## 개발 우선순위

### 1. 문서와 DB 기반 정리

- 교과목 적용 매핑 문서 완성
- ERD 정리
- `item`, `item_alias` 설계
- 주요 조회 인덱스 추가
- README 출품용 소개 초안 작성

### 2. 보안과 사용자 계정

- 회원가입/로그인 API 추가
- BCrypt 비밀번호 저장
- JWT 인증 적용
- 사용자별 데이터 접근 제한 테스트
- secret 관리 방식 정리

### 3. 추천 알고리즘 고도화

- 추천 점수 공식 개선
- 유통기한 임박 재료 우선순위 반영
- 부족 재료/임박 재료 설명 추가
- 추천 결과 UI에 추천 사유 표시

### 4. 이미지 스캔과 품질 개선

- 이미지 리사이즈/압축 적용
- 업로드 제한과 MIME 검증
- 스캔 결과 수정/확정 흐름 다듬기
- 처리 시간/파일 크기 비교 기록

### 5. 인프라와 운영 안정화

- API/DB/reverse proxy Docker Compose 구성
- PostgreSQL 백업 스크립트 작성
- Actuator 상태 확인
- Prometheus/Grafana 또는 최소 상태 대시보드 구성

### 6. 발표/시연 완성

- 시연 데이터 정리
- 발표 자료에 과목별 적용 표 삽입
- 기능별 짧은 시연 시나리오 작성
- 테스트, 빌드, 실행 문서 점검

## 출품 시 강조 메시지

Cibono는 단순한 식재료 관리 앱이 아니라, 데이터베이스 정규화와 인덱스, 자료구조 기반 추천 알고리즘, 이미지 인식, 인증/인가, 컨테이너 기반 운영을 하나의 생활 밀착형 서비스에 통합한 프로젝트이다.
컴퓨터과학과 교과목에서 배운 개념을 실제 문제 해결 과정에 적용했다는 점을 발표와 문서에서 반복적으로 보여준다.
